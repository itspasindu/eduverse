02 — PROBLEM STATEMENT
================================================================================

The Problem
-----------
Students, exam preppers, and informal learners experience low engagement when
studying from textbooks, PDFs, and lecture slides alone. They read passively,
forget quickly, and rarely revisit material in a format that matches how they
actually consume content (short video, humor, social proof). Content creators and
teachers who want to repackage lessons for digital audiences face the same gap:
turning raw notes into polished, persona-driven media is slow and requires
multiple tools (design, TTS, video editing, hosting).

If left unsolved, learners default to passive reading or fragmented apps (chatbot
here, Canva there, YouTube elsewhere) with no single place to study, create, and
share. Teachers cannot easily push class updates or branded mascots alongside
student work.

Why It Matters
--------------
The problem is daily and high-impact for anyone in formal or self-directed
education. Poor engagement leads to lower retention, weaker outcomes, and
abandonment of online learning platforms.

Current workarounds include: generic ChatGPT for Q&A (no persona, no materials
grounding, no video output); meme generators without pedagogy; video tools without
AI scripting from uploads; and LMS forums without creative AI studios. Each
falls short because it solves one slice (Q&A OR memes OR video) without social
context, role-based classrooms, or a unified multimodal pipeline tied to the
user's own uploads and characters.

Root Cause
----------
Study content is still delivered primarily as static text and slides, while
attention and sharing behavior have shifted to visual, short-form, and
persona-driven media. The underlying cause is a toolchain gap: no integrated
system connects (1) the learner's source material, (2) multimodal AI generation,
(3) persistent user identity and library, and (4) community distribution — at a
price point accessible to individuals and small institutions.

03 — PROPOSED SOLUTION
================================================================================

What the Product Does
---------------------
EduVerse is a web application where users register with email/password (Supabase
Auth), complete onboarding (role selection, plan choice), and enter a role-aware
dashboard. The AI Orchestrator centralizes all fal.ai calls: memes, tutoring,
presentations, character reference art, lesson scripts, TTS, image-to-video, and
local ffmpeg muxing. The Learning Agent adds a ReAct loop on top — the tutor can
invoke calculator, code review, and web search tools before answering. Lesson
Studio accepts uploaded materials, pairs them with a custom character (name,
personality, visual description, voice style), and runs a background job that
produces a single combined lesson MP4 with narration. Users publish memes and
videos to a social feed, browse others' posts, and manage a personal library.
Teachers post class announcements; admins moderate users, posts, and subscriptions.

Core mechanism: authenticated API requests flow from Next.js → FastAPI (JWT from
Supabase) → service modules → Supabase Postgres / Storage and fal.ai → results
stored or streamed back to the UI with job polling for long-running lesson videos.

Key Features (built, working, demonstrable)
-------------------------------------------
1. Authentication & profiles (Supabase Auth, JWT middleware, role on signup)
   — Need: secure identity and role-based experiences.

2. Role-aware dashboard (student, creator, teacher, admin navigation)
   — Need: each persona sees relevant tools without clutter.

3. AI Meme Studio (/ai/meme → Flux image + LLM captions)
   — Need: turn study ideas into shareable, humorous visual memory aids.

4. AI Tutor / Learning Agent (/ai/agent/chat, /ai/tutor — ReAct + tools)
   — Need: explanations in standard, simple, or meme modes with optional character
     persona and uploaded material context.

5. Character Designer (lesson_characters table + fal reference image)
   — Need: personalized mascot for tutoring and lesson narration.

6. Lesson Studio (material upload, /ai/lesson-video job pipeline)
   — Need: convert PDF/text materials into ~1–2 min narrated video lessons.

7. Slide Studio (/ai/presentation — creator/admin)
   — Need: generate presentation slides from notes with font styling.

8. Community feed & library (posts, likes, save to library)
   — Need: social motivation and content reuse.

9. Teacher announcements & class mascots
   — Need: classroom communication and shared character visibility.

10. Admin panel (users, moderation, subscription management, analytics overview)
    — Need: platform governance and manual plan assignment.

11. Subscriptions & pricing (Starter free, Pro $12/mo, Institution $29/mo)
    — Need: monetization and tiered feature access narrative.

12. PayHere checkout (sandbox) for paid plans with notify/return/cancel webhooks
    — Need: real payment flow for Pro and Institution tiers.

Scope — In scope for this build

05 — NON-FUNCTIONAL REQUIREMENTS
================================================================================

Performance
-----------
- Meme generation and tutor chat: target < 30s for typical prompts (fal timeout
  default 120s for heavy models).
- Lesson video job: async background task; user polls GET /ai/lesson-video/{id};
  full pipeline may take 2–5+ minutes depending on fal video + ffmpeg mux
  (LESSON_VIDEO_TIMEOUT default 180s).
- Feed and dashboard loads: target < 2s on local/staging with Supabase queries.
- Concurrent usage: stateless FastAPI workers; fal.ai and Supabase handle scale
  externally; lesson jobs are per-user file-based on disk (LESSON_DATA_DIR).

Reliability & Error Handling
----------------------------
- Global FastAPI exception handler returns JSON 500 (preserves CORS headers).
- fal mock mode (FAL_MOCK_MODE=true) provides deterministic placeholders without
  API keys for demos.
- AI routes return 502 with detail message on orchestrator/agent failures.
- Lesson jobs record status pending → processing → completed/failed with
  error_message in lesson_video_jobs table.
- Missing subscription tables degrade gracefully with 503 and migration hint.
- PayHere notify verifies cryptographic hash before activating subscription.

Usability
---------
- Tailwind CSS 4, violet/indigo gradient brand, dark mode support in globals.
- Role-specific sidebar and mobile nav (dashboard-nav.ts).
- Clear CTAs on landing: Start free, View pricing, Sign in.
- Lesson Studio shows job progress, video player with auth-aware blob fetch.
- Pricing cards highlight "Pro" as most popular.

Accessibility
-------------
- Semantic HTML structure in Next.js pages; form labels on auth and settings.
- Further WCAG audit (ARIA live regions for tutor streaming) — future work.

Scalability
-----------
- To handle 10x load: move lesson renders to object storage + CDN; queue lesson
  jobs (Redis/SQS); horizontal scale FastAPI behind load balancer; connection
  pooling on Supabase; rate-limit AI endpoints per plan.
- Architecture already separates concerns (auth, content, ai, subscriptions,
  payments) as independent FastAPI routers for independent scaling.

Data Flow — Lesson Video (core action)
--------------------------------------
1. User uploads file → POST /content/materials → stored in Supabase Storage,
   text extracted (pypdf for PDF) → lesson_materials row.
2. User selects character + material in Lesson Studio → POST /ai/lesson-video.
3. Backend creates lesson_video_jobs row (pending), returns 202 + job_id.
4. Background task run_lesson_video_job:
   a. LLM (Gemini via fal any-llm) writes JSON script: title, narration, visual_prompt.
   b. fal Flux generates scene image (character reference URL if set).
   c. fal TTS generates narration audio (Minimax or Kokoro model).
   d. Optional: fal image-to-video animates scene; ffmpeg muxes audio+video.
   e. Job row updated: progress %, scenes_json, status completed/failed.
5. Frontend polls GET /ai/lesson-video/{job_id} until completed.
6. User plays video via GET /ai/lesson-video/{id}/file or Next.js proxy route
   /api/lesson-video/[jobId]/file with Authorization header.

Data Flow — AI Tutor chat
-------------------------
1. User sends message in TutorChat → POST /api/tutor (Next) or backend /ai/agent/chat.
2. Backend loads optional character persona + material text chunks.
3. EduVerseLearningAgent runs ReAct loop (max AGENT_MAX_TURNS=6):
   - LLM returns JSON action: answer | tool
   - Tools: calculator, code_helper, search (Tavily/Serper HTTP)
4. Final answer returned with steps[] for UI transparency.

AI Integration
--------------
| Capability        | Model / Endpoint                          | Role in UX        |
|-------------------|-------------------------------------------|-------------------|
| Tutor / Agent     | fal-ai/any-llm + gemini-2.0-flash-001   | Q&A, ReAct, tools |
| Meme image        | fal-ai/flux/schnell                       | Meme visuals      |

Prompting: TUTOR_MODE_PROMPTS for standard/simple/meme; build_agent_system_prompt
for tool-use JSON; SCRIPT_SYSTEM for lesson narration JSON-only output.

Known Technical Limitations
---------------------------
- Lesson videos stored on backend filesystem (not Supabase Storage for final MP4).
- Long jobs block background worker per process (no distributed queue).
- Pinecone mentioned in README but not wired in code.
- FAL_MOCK_MODE returns placeholder media for offline demos.
- ffmpeg required on server PATH for mux; gracefully degrades if unavailable.
- PayHere: sandbox credentials in .env.example — must not commit live secrets.
- PDF extraction quality varies; empty text blocks lesson generation.


================================================================================
07 — SECURITY
================================================================================

Authentication & Authorisation
------------------------------
- Users authenticate via Supabase Auth (email/password).
- Frontend: @supabase/ssr cookies; middleware protects /dashboard/*.
- Backend: JWTAuthMiddleware validates Supabase JWT (SUPABASE_JWT_SECRET);
  attaches user_id and user_role to request.state.
- Role enforcement: require_roles() dependency on admin, teacher, creator routes.
- Public paths explicitly listed in config (health, feed, meme demo, plans,
  PayHere webhooks, OpenAPI docs).

Data Handling
-------------
- Profiles, posts, characters, materials, jobs, subscriptions in Supabase Postgres.
- Row Level Security enabled on profiles, posts, subscription tables, lesson tables.
- Passwords managed by Supabase Auth (not stored in application tables).
- Lesson materials and avatars in Supabase Storage buckets (migrations 006, 008).
- Sensitive payment state: PayHere notify verified with merchant secret hash.

API & Secret Management
-----------------------
- Secrets in backend/.env and frontend/.env.local (gitignored).
- Only NEXT_PUBLIC_* exposed to browser (Supabase URL/anon key, API URL).
- FAL_KEY, service role key, JWT secret, PayHere secret — backend only.
- .env.example documents keys without real production values.

Input Validation
----------------
- Pydantic models on all POST bodies (min/max length on prompts, messages).
- Calculator tool uses AST parsing with allowlisted operators only (no eval()).
- File uploads validated by content type and ownership checks on materials.
- PayHere notify payload signature verification before subscription activation.

Known Vulnerabilities or Gaps
-----------------------------
- Subscription tier limits not enforced on all AI routes yet (business logic gap).
- Service role key used server-side; must never ship to client bundles.
- Rate limiting and abuse detection not implemented on /ai/meme public endpoint.
- Content moderation is admin manual review, not automated.
- CORS allows local network regex for dev convenience — tighten for production.

Primary Use Case Walkthrough — Lesson video from upload
-------------------------------------------------------
1. User signs in → lands on dashboard → opens Lesson Studio.
2. User uploads "Biology_Chapter3.pdf" → material appears in list with extracted text.
3. User opens Character Designer (or picks existing) → "Professor Owl", friendly voice.
4. User selects material + character → clicks Generate Lesson Video.
5. System creates job, shows progress bar; backend runs script → image → TTS → mux.
6. On completion, inline video player loads MP4 via authenticated proxy.
7. User optionally saves output to library or shares via feed (meme/video post types).

Edge Cases
----------
- Empty PDF text → 400 with message to use .txt/.md or re-upload.
- fal API down → 502 on sync routes; lesson job status failed with error_message.
- User polls job belonging to another user → 404.
- Free plan selected → POST /subscriptions/me without PayHere.
- Paid plan without PayHere env → 503 with configuration instructions.
- Unauthenticated dashboard access → redirect to /login?redirect=...
- Agent JSON parse failure → falls back to raw LLM text answer.
